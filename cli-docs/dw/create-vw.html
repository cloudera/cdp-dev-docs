create-vw¶

Description¶

Create a Virtual Warehouse.

Synopsis¶

  create-vw
--cluster-id <value>
--dbc-id <value>
--vw-type <value>
--name <value>
[--template <value>]
[--autoscaling <value>]
[--cli-input-json <value>]
[--generate-cli-skeleton]

Options¶

--cluster-id (string) ID of cluster where Virtual Warehouse should be created.

--dbc-id (string) ID of Database Catalog that the Virtual Warehouse should be attached to.

--vw-type (string) Type of Virtual Warehouse: ‘hive’ or ‘impala’.

Possible values:

hive impala

--name (string) Name of the Virtual Warehouse.

--template (string) Name of configuration template to use.

Possible values:

xsmall small medium large

--autoscaling (object) Configurations for auto-scaling a Virtual Warehouse.

Shorthand Syntax:

minClusters=integer,maxClusters=integer

JSON Syntax:

{
  "minClusters": integer,
  "maxClusters": integer
}

--cli-input-json (string) Performs service operation based on the JSON string provided. The JSON string follows the format provided by --generate-cli-skeleton. If other arguments are provided on the command line, the CLI values will override the JSON-provided values.

--generate-cli-skeleton (boolean) Prints a sample input JSON to standard output. Note the specified operation is not run if this argument is specified. The sample input can be used as an argument for --cli-input-json.

Output¶

vwId -> (string)

ID of the new Virtual Warehouse.

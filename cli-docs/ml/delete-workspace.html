delete-workspace¶

subtitle

Delete Cloudera Machine Learning Workspace.

version

0.9.47

Description¶

Deletes a Cloudera Machine Learning workspace.

Synopsis¶

  delete-workspace
--force | --no-force
[--remove-storage | --no-remove-storage]
[--environment-name <value>]
[--workspace-name <value>]
[--workspace-crn <value>]
[--cli-input-json <value>]
[--generate-cli-skeleton]

Options¶

--force | --no-force (boolean) Force delete a workspace even if errors occur during deletion. Force delete removes the guarantee that resources in your cloud account will be cleaned up.

--remove-storage | --no-remove-storage (boolean) The remove storage flag indicates weather to keep the backing workspace filesystem storage or remove it during delete.

--environment-name (string) The environment for the workspace to delete.

--workspace-name (string) The name of the workspace to delete.

--workspace-crn (string) The CRN of the workspace to delete. If CRN is specified only the CRN is used for identifying the workspace, environment and name arguments are ignored.

--cli-input-json (string) Performs service operation based on the JSON string provided. The JSON string follows the format provided by --generate-cli-skeleton. If other arguments are provided on the command line, the CLI values will override the JSON-provided values.

--generate-cli-skeleton (boolean) Prints a sample input JSON to standard output. Note the specified operation is not run if this argument is specified. The sample input can be used as an argument for --cli-input-json.

list-users¶

subtitle

List Users

version

0.9.100

Description¶

Lists users and roles of a compute cluster.

Synopsis¶

  list-users
--cluster-crn <value>
[--only-admins | --no-only-admins]
[--cli-input-json <value>]
[--generate-cli-skeleton]

Options¶

--cluster-crn (string)

Compute cluster CRN.

--only-admins | --no-only-admins (boolean)

Set true to list only admin users.

--cli-input-json (string)

Performs service operation based on the JSON string provided. The JSON string follows the format provided by --generate-cli-skeleton. If other arguments are provided on the command line, the CLI values will override the JSON-provided values.

--generate-cli-skeleton (boolean)

Prints a sample input JSON to standard output. Note the specified operation is not run if this argument is specified. The sample input can be used as an argument for --cli-input-json.

Output¶

users -> (array)

List of Users.

item -> (object)

User details.

groups -> (array)

K8s RBAC cluster role bindings.

item -> (string)

id -> (string)

User ID.

isAdmin -> (boolean)

True if added user is an admin.

isRole -> (boolean)

True if user ID is a role.

username -> (string)

Username.

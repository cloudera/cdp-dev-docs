upgrade-database¶

subtitle

Upgrades the database

version

0.9.46 (BETA)

Description¶

Upgrades an operational database in an environment to a given runtime.

Synopsis¶

  upgrade-database
--environment <value>
--database <value>
[--runtime <value>]
[--os-upgrade-only | --no-os-upgrade-only]
[--cli-input-json <value>]
[--generate-cli-skeleton]

Options¶

--environment (string) The name or crn of the environment.

--database (string) The name or crn of the database.

--runtime (string) Target runtime for an upgrade.

--os-upgrade-only | --no-os-upgrade-only (boolean) Only perform an Operating System upgrade.

--cli-input-json (string) Performs service operation based on the JSON string provided. The JSON string follows the format provided by --generate-cli-skeleton. If other arguments are provided on the command line, the CLI values will override the JSON-provided values.

--generate-cli-skeleton (boolean) Prints a sample input JSON to standard output. Note the specified operation is not run if this argument is specified. The sample input can be used as an argument for --cli-input-json.

Output¶

statusReason -> (string)

The reason whether upgrade request is accepted or why it is not possible.
